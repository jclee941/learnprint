## 한 문장 요약

흩어진 전공 학습 흔적을 AI로 "역량 언어"로 번역하고, **결정론적 규칙 엔진이 6개 역량(+기타)으로 자동 분류해 증거 기반 학습 이력서로 재구성**하는 웹앱과, 그 위에서 한 학기 6개 과목에 적용한 **"설명 → 진단 → 증거" 3-루프 학습법**.

## 문제 (Why)

한 학기에 여러 과목을 들으면 학습 흔적이 LMS·노트·과제 폴더·코드 저장소에 흩어진다. "수강 완료"는 남지만 "이 과목에서 무엇을 이해했고 어디에 쓸 수 있는가"를 한 문장으로 설명하지 못한다. 머릿속에 학습은 있으나 **이력서 언어로 번역되지 못한 채 잠들어 있는** 상태가 1~3학년 내내 가장 큰 문제였다.

## 학습법 (How) — 설명·진단·증거 3-루프

| 루프 | 핵심 행동 | AI의 역할 |
|------|-----------|-----------|
| 1. 설명 | 개념을 내 말로 한 단락 적는다 (모르면 모른다고 적음) | 사용 안 함 — 먼저 내 빈틈을 가른다 |
| 2. 진단 | 내 요약을 AI에 보여주고 빈틈·오류·빠진 예시를 짚게 함 | **답안 생성기가 아니라 진단기** — 받아들일/거부할 부분을 직접 표시 |
| 3. 증거 | 받아들인 부분만 다시 적고 learnprint에 흔적 1건으로 기록 | 규칙 엔진이 6개 역량(+기타)으로 자동 분류 |

핵심 프레이밍: **AI를 "답안 생성기"가 아니라 "내 학습 흔적을 역량 언어로 바꾸는 도구"로 사용**. 한 단원의 루프가 끝나면 다음 단원에서 "어느 부분이 비어있는지"를 시작부터 안다.

## 도구 (What) — learnprint

- **입력**: 강의·과제·프로젝트·자격증·독학·대회 등 학습 흔적을 증거(링크·메모)와 함께 기록
- **자동 분류**: 키워드 매칭 기반 **결정론적 엔진**이 6개 역량(데이터·AI 활용 / 문제해결·논리력 / 협업·커뮤니케이션 / 자기주도학습 / 전공·직무역량 / 기획·문서작성)으로 분류, 안 잡히면 "기타 학습 경험"
- **내보내기·인쇄**: Markdown / JSON / 증거 원장(Evidence Ledger) + 브라우저 인쇄
- **(선택) AI 에이전트**: 자기소개 작성·학습 코칭·면접질문 생성 — AI 없이도 핵심 기능 완전 동작

## 차별점 (참신성)

1. **AI = 역량 언어 변환기** (답안 생성기 아님) — 학습 행동 자체를 바꾸는 프레이밍
2. **설명 가능한 결정론 엔진** — 동일 입력 → 동일 결과, 심사·재현 시 결과가 흔들리지 않음
3. **결정론 엔진 + LLM 이중화** — 네트워크·AI 없이도 동작하고, AI는 그 위에 얹는 선택 레이어
4. **증거 원장(Evidence Ledger)** — 어떤 학습이 어떤 역량에 묶이는지 출처와 함께 추적
