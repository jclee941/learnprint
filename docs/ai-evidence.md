## 3. 학습법에서의 AI 활용: 설명·진단·증거 3-루프

사례보고서 §2에서 설명한 3-루프 학습법은 (1) 강의를 내 언어로 요약하고 (2) 이해가 안 가는 부분을 AI에게 구체적으로 질문해 개념을 재구성하고 (3) 그 결과물(질문 로그·오답 노트·성찰문)을 역량별로 분류해 저장하는 것이다. 아래 다섯 개 프롬프트는 6개 과목에서 실제로 재사용한 템플릿이다. `[대괄호]` 안만 바꾸어 모든 과목에 그대로 적용했다.

### 3.1 설명 검증 · 추상 개념을 비유·표·시각화로 재구성

```
[개념 이름]을 [대상 수준: 비전공자 / 후배 / 초등학생]에게 설명하듯
[형식: 비유 / 비교표 / 단계별 그림]으로 알려줘.
[사례 1], [사례 2], [사례 3]에서 이 개념이 어떻게 나타나는지도 함께 정리해줘.
설명 마지막에 내가 이해했는지 확인할 수 있는 질문 2개도 던져줘.
```

적용 사례: 전기전자정보통신개론(진폭·주파수 비유), 모바일시스템(생명주기 비교표), 컴퓨터그래픽스와VR(렌더링 파이프라인 단계별).

### 3.2 진단 · 증상에서 원인과 해결 순서 뽑기

```
[증상 또는 현상]이 발생했을 때 일반적인 원인을 우선순위 순서로 알려줘.
각 원인에 대해 [언어/엔진/시스템]에서 어떻게 검증하는지도 알려줘.
마지막에 같은 증상이 재발하지 않게 만드는 예방 코드 패턴이나 점검 항목을 줘.
```

적용 사례: 게임프로그래밍개발(2D 플랫폼에서 캐릭터가 벽을 뚫는 버그), 모바일시스템(저장소 용량 초과로 인한 데이터 손실).

### 3.3 퀴즈 · 개념 이해도와 적용력을 번갈아 검증

```
[주제]를 퀴즈 형식으로 5문제 출제해줘.
[짝수]번은 개념 이해 문제, [홀수]번은 적용 문제로 구성해줘.
각 문제에 정답과 해설, 그리고 내가 자주 틀릴 만한 오답 포인트를 함께 적어줘.
```

적용 사례: 컴퓨터구조론(캐시·파이프라인·메모리 계층 5문제), 전기전자정보통신개론(신호·회로 5문제).

### 3.4 오답 수정 · 틀린 부분을 내 말로 다시 쓰기

```
내가 적은 답은 다음과 같아: "[내 답변]".
이 답이 [개념]의 어느 부분에서 틀렸는지 짚어주고, 오답 노트 형태로
(1) 내가 빠뜨린 핵심 포인트
(2) 다시 외울 한 문장 정의
(3) 다음에 같은 문제가 나와 적용할 점검 절차
세 항목으로 정리해줘.
```

적용 사례: 컴퓨터구조론 오답 노트, 전기전자정보통신개론 진폭/주파수 적용 문제 오답.

### 3.5 역량 증거 변환 · 학습 흔적을 역량별 증거 문장으로

```
다음은 내가 한 학습 활동이야:
- 제목: [제목]
- 유형: [강의/과제/프로젝트/독학/자격증/기타]
- 기간: [학기/날짜]
- 설명: [내가 한 일을 한두 문장]
- 증거: [링크 또는 산출물]

이 학습 활동을 다음 6개 역량 중 가장 어울리는 하나로 분류하고,
자기소개서나 면접에서 바로 인용할 수 있는 한 문장 증거를 만들어줘.
역량 목록: 데이터·AI 활용, 문제해결·논리력, 협업·커뮤니케이션,
자기주도학습, 전공·직무역량, 기획·문서작성.
```

적용 사례: 사례보고서 §3의 6개 과목 항목 설명을 learnprint 입력 폼에 그대로 옮길 때 이 템플릿으로 자기 점검 후 입력했다.

## 4. 대표 프롬프트 및 활용 결과 기록 (코딩)

도구 제작 측면에서 실제로 AI에게 던졌던 프롬프트 중 결정적인 네 개를 기록한다. 각 예시는 (프롬프트) → (AI 결과 요약) → (본인 검토·수정) → (최종 반영) 순으로 적었다.

### 예시 1. 결정론적 분석 엔진 구현

**프롬프트 (사용자 입력)**

> 학습 경험을 역량별로 분류하는 결정론적 규칙 기반 엔진을 TypeScript로 TDD 방식으로 구현해줘. 입력은 LearningItem 배열이고, 출력은 6개 역량(데이터·AI 활용, 문제해결·논리력, 협업·커뮤니케이션, 자기주도학습, 전공·직무역량, 기획·문서작성) + 기타로 그룹화해야 해. 키워드 매칭 방식으로 결정론적으로 동작해야 하고, 네트워크 호출은 절대 하면 안 돼.

**AI 결과 요약**

AI는 `LearningAnalyzer` 인터페이스와 `analyzeLearningItems` 함수를 제안했다. 각 역량에 키워드 배열을 할당하고, 입력 텍스트에서 키워드 매칭 개수로 점수를 계산한 뒤 최고점 역량에 배당하는 구조였다.

**본인 검토·수정 내용**

- AI가 제안한 초안 키워드는 영어 중심(`data`, `ml`, `python` 등)이라 한국어 학습 맥락에 약했다. 키워드 사전을 직접 재구성해 "비교표", "도식화", "셰이더", "엔진", "포그라운드", "퀴즈 형식" 같은 한국어 학습 어휘를 보강했다.
- "기타 학습 경험" 폴백(fallback) 처리와 빈 입력 가드, `isEmpty` / `generatedAt` 메타데이터 필드를 추가해 UI에서 상태를 명확히 표현하도록 했다.
- 네트워크 미사용을 명시적으로 검증하는 테스트(`analyzer:uses-no-network-or-secrets`)를 별도로 추가했다.

**최종 반영 결과**

- `src/lib/ai/analyzer.ts` 구현 완료
- `src/lib/ai/analyzer.test.ts` 단위 테스트 통과 (역량 분류 결정론성, 빈 입력, 네트워크 미사용 검증)

### 예시 2. localStorage 저장소 계층 구현

**프롬프트 (사용자 입력)**

> React 앱에서 localStorage를 사용해 LearningItem 배열을 저장·로드·삭제하는 저장소 계층을 TypeScript로 구현해줘. JSON 파싱 에러가 나도 앱이 죽지 않도록 복구 로직을 넣고, TDD로 테스트부터 작성해줘.

**AI 결과 요약**

AI는 `saveLearningItems`, `loadLearningItems`, `clearLearningItems` 함수와 `STORAGE_KEY` 상수를 제안했다. JSON 파싱 실패 시 빈 배열을 반환하는 try/catch 복구 패턴을 포함했다.

**본인 검토·수정 내용**

- 저장소 키 네이밍을 `hycu-learning-resume:items:v1` 형태로 버전을 명시해 두어, 추후 스키마 변경 시 마이그레이션 단서를 남겼다.

- 저장소 계층이 네트워크·환경변수를 사용하지 않는다는 테스트(`storage:does-not-require-network-or-env`)를 추가해 민감 정보 누수 경로를 잠갔다.

**최종 반영 결과**

- `src/lib/storage/storage.ts` 구현 완료

### 예시 4. 역량 증거 원장 exporter (TDD)

**프롬프트 (사용자 입력)**

> 학습 경험을 역량별 증거 원장 Markdown 표로 내보내는 결정론적 exporter를 TDD로 구현해줘. 표는 한 행 = 한 학습 흔적이고, 열은 (학습 경험 / 유형·기간 / 인공물·출처 / 역량 / 증거 문장 / 한계) 6개로 해줘. 각 행에 행별 한계 셀을 넣고, 마지막에 전체 한계 주석(성적·출석·LMS 원본을 자동 검증하지 않음)도 항상 포함해줘. 같은 입력에 대해 항상 같은 문자열이 나와야 해. 파일명은 `learning-evidence-ledger.md`로 고정해줘.

**AI 결과 요약**

AI는 `resumeToEvidenceLedgerMarkdown` 함수, Markdown 표 빌더, `escapeTableCell` 헬퍼(`|`와 줄바꿈 이스케이프), 그리고 표 머리·행·한계·인공물 출처·결정론·링크 렌더·철자를 검증하는 테스트 케이스를 함께 제안했다.

**본인 검토·수정 내용**

- AI 초안은 `Date.now()`를 직접 호출해 결정론을 깰 위험이 있었다. `resume.generatedAt`만 사용해 동일 입력 동일 출력을 보장하도록 수정했다.
- AI 초안의 표는 4개 열(학습 경험/유형·기간/역량/증거 문장)만 가져서, "증거가 어디에 남아 있는가"와 "이 행의 한계"를 드러내지 못했다. 그래서 `인공물·출처`(증거 링크가 있으면 파일·링크, 없으면 "기록 본문")과 행별 `한계` 열을 직접 추가해, 한 행만 봐도 학습·인공물·역량·한계가 한눈에 읽히게 했다.
- 표 셀 이스케이프(`|`, `\n`) 누락 시 Markdown이 깨지는 문제를 발견하고 `escapeTableCell`을 강제 적용했다.
- 전체 한계 주석을 `> 한계:` 인용구로 항상 마지막 줄에 넣어, 사용자가 "이 도구가 무엇을 검증하지 않는지" 출력물 자체에서 바로 읽을 수 있게 했다. 출력 흔적의 도입부 하단(설명)에 있던 철자 오류(흔어진 -> 흩어진)도 바로잡았다.
- 결정론 보존을 위해 정렬·반복·객체 키 순서를 모두 입력(`resume`)에 의존하도록 강제했다.

**최종 반영 결과**

- `src/features/resume/exporters.ts`에 `resumeToEvidenceLedgerMarkdown` 구현
- `src/features/resume/exporters.test.ts`의 Evidence Ledger 테스트 6건(표 머리 6열·행별·전체 한계·인공물 출처·결정론·링크 렌더·철자) + `ExportControls.test.tsx`의 버튼 다운로드 테스트 1건 통과

## 5. 인간 검토·수정 기록 (Human Review & Correction)

AI가 제안한 결과를 그대로 쓴 적은 한 번도 없다. 아래는 실제 검토 과정에서 "AI의 초안이 부족하거나 틀렸다고 판단해 내가 고친" 대표 사례다.

| # | 상황 | AI가 준 결과 | 부족했던 점·오류 | 본인이 수정·검증한 내용 |
|---|------|--------------|-----------------|------------------------|
| 1 | 분석 엔진 초안 (예시 1) | 영어 중심 키워드 사전(`data`, `ml`, `python` 등) | 한국어 학습 어휘가 거의 잡히지 않음. "셰이더", "엔진", "비교표", "도식화" 같은 과목 표현이 누락 | 키워드 사전을 한국어 학습 어휘로 재구성하고, "기타 학습 경험" 폴백·빈 입력 가드·메타데이터 필드를 추가함. 네트워크 미사용 테스트를 별도로 작성해 잠금 |
| 2 | 전기전자정보통신개론 학습 (사례보고서 §3.4) | 진폭·주파수를 "진폭은 높이, 주파수는 빠르기" 한 줄로 단순 설명 | 너무 일반적. 소리·라디오·Wi-Fi 같은 구체적 맥락이 없음 | "소리, 라디오, Wi-Fi 신호 각각에서 어떻게 나타나는지"를 보강해 다시 질문했고, 비유와 사례를 받아 도식화 노트로 직접 재구성함 |
| 3 | 컴퓨터구조론 학습 (사례보고서 §3.5) | 캐시와 파이프라인을 각각 별개 항목으로만 설명 | 두 개념이 어떻게 맞물려 CPU 내부 흐름을 만드는지 보여주지 못함 | "CPU 내부에서 명령어가 처리되는 전체 흐름"을 하나의 이야기로 다시 묶어달라고 재질문하고, 내 말로 다시 그림을 그려 복습함 |
| 4 | 증거 원장 exporter (예시 4) | `Date.now()` 호출 + 정렬 미고정으로 동일 입력 비동일 출력 위험 | 결정론 위반. 표 셀이 `|`·줄바꿈 포함 시 깨짐 | `resume.generatedAt`만 사용하도록 교체하고 `escapeTableCell`로 안전하게 감쌈. 한계 주석을 `> 한계:` 인용구로 항상 출력하도록 강제 |
| 5 | AI 에이전트 응답 검토 | "모바일 앱 생명주기를 표로 정리해줘" 답변을 그대로 사용 가능 상태로 받음 | 정답은 맞지만 내 학습 노트와 어조가 다름 | 표를 받아 주차별 학습노트로 내 말로 재작성하고, SharedPreferences vs Room 등 차이점은 본인이 정리함 |

## 7. 과대표현 금지 경계 (No Over-Claim Boundary)

평가의 진실성을 위해 본 도구가 **하지 않는** 것을 명시한다.

- **LMS 자동 검증 안 함.** HYCU LMS(lms.hycu.ac.kr) 출석·진도·성적·수강 이력을 자동으로 읽거나 검증하지 않는다. 사용자가 입력한 학습 경험과 Evidence 필드만 다룬다. Evidence Ledger 출력에도 `> 한계: ... 성적·출석·LMS 원본을 자동 검증하지 않습니다` 주석이 항상 포함된다.
- **성적·학점 예측 안 함.** AI 에이전트(coach, interview, competency 모드)도 성적 예측·학점 산정·수강 추천을 수행하지 않는다. 학습 코칭과 면접 질문 생성으로만 동작한다.
- **의미론적 AI 채점 안 함.** 분석은 결정론적 키워드 매칭(`analyzeLearningItems`)으로만 동작한다. LLM 에이전트는 사용자의 학습 이력을 자연어로 정리·요약할 뿐, "이 답이 몇 점이다" 형태의 의미론적 채점을 하지 않는다.
- **자격증·인증 자동 발급 안 함.** 자격증·인증·수료증의 진위를 자동 확인하지 않으며, 자격증 번호나 발급기관 정보를 다루지 않는다.
- **학번·인증정보·비밀키 입력·저장·노출 안 함.** 학습 데이터는 브라우저 localStorage에만 저장되며, 사용자는 학번·비밀번호·인증 토큰을 입력하지 않는 것을 원칙으로 한다. 설정값·비밀키는 저장소에 커밋되지 않는다(`.gitignore`로 `.env` 차단).
- **AI 답변을 무조건 신뢰하지 않음.** §5에서 보듯 AI 초안은 항상 본인이 검토·수정·재작성한다. 전기전자정보통신개론의 한 줄 단순 설명 사례처럼, AI 답변이 학습 목적에 부족하면 반드시 재질문하고 내 말로 다시 쓴다.
- **외부인 평가·랭킹·비교 안 함.** 다른 학생과의 비교·순위·우수 수상 이력 같은 정보는 다루지 않는다.
