# 2026학년도 HYCU AI 학습법 사례보고서

## 1. 내가 겪은 전공 학습 문제

나는 한양사이버대학교 컴퓨터 전공 학생이다. 매 학기 여러 과목을 수강하지만, 각 과목의 학습 내용이 LMS, 개인 노트, 과제 제출함, 프로젝트 폴더 등 여기저기 흩어져 있었다. "이 과목 수강 완료했다"는 사실은 남아도, "무엇을 이해하고 어떻게 적용했는지"를 구체적으로 설명하라고 하면 막막했다.

특히 게임프로그래밍개발이나 컴퓨터구조론 같은 과목에서는 코드를 직접 써보고 버그도 잡아봤는데, 면접이나 자기소개서에서는 "강의 들었다"는 한 마디로 끝나버렸다. 모바일시스템이나 전기전자정보통신개론 같은 이론 과목은 개념이 추상적이라 복습 후에도 머릿속에 체계가 잡히지 않았다. 사랑의실천 같은 교양 과목조차도 봉사 경험을 개발자의 윤리적 책임과 연결 지어 글로 남기기 어려웠다.

결국 내 머릿속에 있는 학습 경험을 "역량 언어"로 바꿔서 증거와 함께 제시할 수 있는 방법이 필요했다. 그래서 나는 AI를 단순 답안 생성기가 아닌, 질문 설계와 개념 재구성, 퀴즈와 회고를 돕는 도구로 삼는 학습법을 만들었고, 이 과정을 기록하고 검증할 수 있는 learnprint라는 웹앱을 직접 제작했다.

## 2. 내가 만든 AI 학습법: 과목 기록을 역량 증거로

나의 핵심 학습법은 세 단계로 요약할 수 있다. 첫째, 강의나 실습 후에 내 언어로 요약한다. 둘째, 이해가 안 가는 부분을 AI에게 구체적으로 질문해서 개념을 재구성한다. 셋째, 그 과정에서 나온 질문 로그, 오답 노트, 개념도, 성찰문 등을 역량별로 분류해서 저장한다.

AI에게는 이런 식으로 질문한다. "모바일 앱의 생명주기를 데이터 저장 관점에서 비교표로 정리해줘", "캐시 메모리와 파이프라인 개념을 퀴즈 5문제로 내줘", "렌더링 파이프라인을 비전공자에게 설명하듯 시각적으로 설명해줘". 답변을 그대로 베끼는 게 아니라, 틀린 부분은 오답 노트로 만들고, 이해한 부분은 내 말로 다시 써본다.

learnprint는 이 학습법을 기록하고 검증하는 증거 도구다. 6개 과목의 학습 기록을 입력하면, 규칙 기반 분석 엔진이 역량별로 그룹화된 이력서를 만든다. 네트워크가 없을 때도 이 오프라인 폴백이 작동해서 학습 데이터가 사라지지 않는다. 또한 LLM 기반 AI 에이전트를 연결하면, 등록한 학습 이력을 바탕으로 역량 분석, 학습 코칭, 예상 면접 질문까지 받을 수 있다. AI 에이전트는 서버 측 BFF를 통해 cliproxy LLM API와 통신하며, API 키는 서버 .env에만 보관되어 브라우저로 노출되지 않는다.

## 3. 과목별 AI 활용 사례

### 3.1 게임프로그래밍개발

**무엇이 어려웠나.** 게임 루프와 충돌 처리를 직접 구현하는 실습에서, 캐릭터가 벽을 뚫고 지나가는 버그가 계속 생겼다. 업데이트 순서가 잘못된 것 같은데 원인을 몰라서 한참을 헤맸다.

**AI에게 어떻게 질문했나.** "2D 플랫폼 게임에서 충돌 처리 후 캐릭터가 벽을 뚫는 버그의 일반적인 원인과 해결 순서를 알려줘"라고 물었다. AI는 업데이트 순서, 델타 타임 적용 시점, 충돌 검사와 위치 보정의 타이밍을 단계별로 제시했다.

**무엇을 만들었나.** AI가 제시한 순서를 따라가며 코드를 수정하고, 간단한 2D 플랫폼 프로토타입을 완성했다. 게임 엔진 내부 동작 원리를 직접 체감하면서, 단순히 엔진을 "쓰는" 사람이 아니라 원리를 "이해하는" 사람이 되었다.

**학습 전후 변화.** Before: 유니티나 Godot 예제를 따라치기만 했다. After: 게임 루프 한 사이클 안에서 무슨 일이 일어나는지 머릿속으로 그릴 수 있게 되었다.

### 3.2 모바일시스템

**무엇이 어려웠나.** 모바일 앱의 생명주기와 포그라운드·백그라운드 전환 시 데이터 저장 방법은 강의로 들을 때는 알 것 같았는데, 실제로 정리하려니 개념이 뒤섞였다. 저장소 용량 제한과 네트워크 불안정 상황에서의 대응 전략도 체계적으로 정리하기 어려웠다.

**AI에게 어떻게 질문했나.** "안드로이드 액티비티 생명주기 각 단계에서 데이터를 저장하고 복원하는 방법을 표로 비교해줘. 저장소 종류와 용량 제한, 네트워크 불안정 대응 전략도 함께 정리해줘"라고 요청했다.

**무엇을 만들었나.** AI가 생성한 초안을 바탕으로 개념 비교표를 직접 다듬었다. SharedPreferences와 Room, 파일 저장, 네트워크 캐싱의 차이점을 내 말로 정리한 주차별 학습노트를 남겼다.

**학습 전후 변화.** Before: "onPause에서 저장하고 onResume에서 불러오면 된다"는 막연한 인식. After: 각 생명주기 단계의 책임과 데이터 손실 위험을 구체적으로 설명할 수 있다.

### 3.3 사랑의실천

**무엇이 어려웠나.** 봉사활동과 윤리적 책임을 개발자라는 전공 시선에서 되돌아보는 게 쉽지 않았다. 성찰문을 쓰려고 해도 어디서 시작해야 할지 막막했다.

**AI에게 어떻게 질문했나.** "봉사활동 경험을 개발자의 사회적 책임과 연결 짓는 성찰문의 구조와 핵심 질문을 제시해줘"라고 물었다. AI는 경험 서술, 문제 인식, 전공 연결, 실천 계획의 4단계 구조를 제안했다.

**무엇을 만들었나.** 실제 본인의 봉사 경험을 바탕으로, 개발자 윤리와 사회적 책임에 대한 성찰문을 직접 썼다. AI가 준 구조는 참고했지만, 모든 사례와 감정은 본인의 것이다.

**학습 전후 변화.** Before: 교양 과목을 그냥 "들어야 하는 과목"으로 여겼다. After: 개발자로서의 정체성과 사회적 역할을 연결해서 생각하게 되었다.

### 3.4 전기전자정보통신개론

**무엇이 어려웠나.** 신호와 회로, 통신 기초 중에서 진폭과 주파수 개념이 처음에는 너무 추상적으로 느껴졌다. 수식은 보이는데, 이것이 실제로 무엇을 의미하는지 상상이 안 됐다.

**AI에게 어떻게 질문했나.** "진폭과 주파수를 일상적 비유와 구체적 예제를 들어서 설명해줘. 소리, 라디오, Wi-Fi 신호 각각에서 이 개념이 어떻게 나타나는지도 함께 알려줘"라고 요청했다.

**무엇을 만들었나.** AI가 제시한 비유와 예제를 바탕으로 개념을 재학습한 뒤, 관련 개념을 직접 도식화해서 정리 노트를 만들었다. LMS 학습 기록과 함께 이 노트를 증거로 남겼다.

**학습 전후 변화.** Before: "진폭은 높이, 주파수는 빠르기"라는 피상적 이해. After: 다양한 신호 처리 상황에서 왜 진폭과 주파수가 중요한지 체계적으로 설명할 수 있다.

### 3.5 컴퓨터구조론

**무엇이 어려웠나.** 캐시 메모리와 파이프라인, 메모리 계층 구조는 각각의 정의는 외울 수 있어도, CPU 내부에서 실제로 어떻게 연결되는지 머릿속에 그리기 어려웠다.

**AI에게 어떻게 질문했나.** "캐시 메모리와 파이프라인, 메모리 계층 구조를 퀴즈 형식으로 5문제 출제해줘. 각 문제는 개념 이해와 적용 문제를 번갈아가며 구성해줘"라고 요청했다.

**무엇을 만들었나.** AI가 낸 퀴즈를 풀고, 틀린 문제는 오답 노트를 직접 만들어 복습했다. 캐시 히트/미스와 파이프라인 버블의 인과관계를 그림으로 정리했고, CPU 내부에서 명령어가 처리되는 전체 흐름을 머릿속으로 그릴 수 있게 되었다.

**학습 전후 변화.** Before: 용어 암기 중심의 단편적 지식. After: CPU-메모리-캐시-파이프라인의 데이터 흐름을 하나의 이야기로 설명할 수 있다.

### 3.6 컴퓨터그래픽스와VR

**무엇이 어려웠나.** 렌더링 파이프라인과 좌표 변환, 셰이더의 역할은 이미지로 보면 멋있는데, 수학적 원리가 복잡해서 머리에 잘 남지 않았다. VR 환경에서의 인터랙션 원리도 막연했다.

**AI에게 어떻게 질문했나.** "렌더링 파이프라인의 각 단계를 시각적으로 설명해줘. 정점 처리, 래스터화, 프래그먼트 처리가 실제로 화면에 어떻게 나타나는지 단계별로 그려줘. 셰이더가 각 단계에서 무슨 역할을 하는지도 함께 알려줘"라고 요청했다.

**무엇을 만들었나.** AI가 시각적으로 설명한 개념을 바탕으로 셰이더 실습 코드를 작성하고, VR 인터랙션 분석 노트를 정리했다. 복잡한 3D 그래픽스 개념을 내 말로 다시 써보니 이해가 훨씬 쉬워졌다.

**학습 전후 변화.** Before: "셰이더는 예쁜 효과를 주는 것"이라는 막연한 인식. After: 3D 공간의 좌표 변환부터 픽셀 색상 결정까지 전체 파이프라인을 단계별로 설명할 수 있다.

## 4. Before / After: AI가 바꾼 학습 행동

**Before:** 나의 학습은 "수강 완료" 중심이었다. 과목 간 연결이 부족했고, 자기소개나 면접에서 근거를 대라고 하면 할 말이 없었다. 복습은 시험 직전에만 했고, 이해했다고 착각한 개념은 다시 돌아보지 않았다.

**After:** AI를 활용한 학습 루틴이 자리 잡았다. 강의 수강 후 요약을 내 언어로 쓴다. 이해가 안 가는 부분은 AI에게 구체적으로 질문해서 개념을 재설명받는다. 그 결과물과 질문 로그를 learnprint에 기록한다. 이력서가 필요할 때는 이 기록을 바탕으로 역량별 증거를 뽑아서 Markdown이나 JSON으로 내보낸다.

가장 큰 변화는 질문의 질이 달라졌다는 것이다. 처음에는 "이거 설명해줘"라는 막연한 질문을 했다면, 지금은 "이 개념을 비유와 예제로 설명하고, 퀴즈로 검증해줘"처럼 단계와 목적이 분명한 질문을 설계한다. AI는 답을 주는 게 아니라, 나에게 더 좋은 질문을 하게 만드는 거울이 되었다.

## 5. 결과물과 검증

learnprint는 내가 6개 과목의 학습 기록을 입력하면, 데이터·AI 활용, 문제해결·논리력, 협업·커뮤니케이션, 자기주도학습, 전공·직무역량, 기획·문서작성이라는 6개 역량 그룹으로 의미 있게 분류한다. 각 역량 아래에는 관련 과목의 제목, 유형, 기간, 요약, 증거 링크가 제시되어 "어떤 학습이 어떤 역량을 뒷받침하는가"를 구체적으로 보여준다.

생성된 이력서는 Markdown 파일과 JSON 파일로 내보낼 수 있어서, 자기소개서나 포트폴리오 작성 때 바로 활용할 수 있다. 학습 데이터는 브라우저의 localStorage에 저장되므로 새로고침 후에도 유지된다.

AI 에이전트 기능은 세 가지를 제공한다. 첫째, 역량 분석과 자기소개 문단 작성. 둘째, 학습 코칭 챗봇. 셋째, 예상 면접 질문 생성. 이 기능은 LLM 에이전트가 서버 측 BFF를 통해 cliproxy LLM API와 통신하여 실시간 스트리밍 응답을 제공한다. 네트워크나 AI 연결이 불안정할 때는 오프라인 규칙 기반 폴백 엔진이 동일한 입력으로 동일한 결과를 만들어준다.

**개인정보·보안:** 학습 데이터는 기본적으로 브라우저 localStorage에만 저장된다. AI 에이전트를 사용할 때는 학습 이력 일부가 서버 측 BFF인 `/api/agent/chat`을 거쳐 LLM API로 전송될 수 있다. API 키는 서버의 `.env` 파일에만 보관되며 브라우저에는 노출되지 않는다. 민감정보인 학번, 인증정보, 비밀번호는 앱에 입력하지 않는다.

## 6. 한계와 보완

AI가 제공하는 답변은 항상 검증이 필요하다. 특히 컴퓨터구조론이나 전기전자정보통신개론 같은 과목에서는 AI가 제시한 개념 설명 중 일부가 부정확할 수 있어서, 교재나 강의 자료와 대조하는 습관을 들여야 한다.

개인정보 보호를 위해 학번이나 인증정보, 비밀번호는 앱에 입력하지 않는다. 앞으로 과목별 과제 회고를 더 축적하고, 틀린 개념을 오답 노트로 남기는 습관을 지속할 계획이다.

규칙 기반 분석 엔진은 키워드 매칭으로 작동하기 때문에, 새로운 역량 영역이나 예외적인 과목 설명에는 한계가 있다. 이 부분은 LLM 기반 AI 에이전트가 보완하는 구조로 설계되어 있다.

## 부록. 검증 결과와 정직성 선언

**검증 결과.** 본 프로젝트는 `npm test`로 40개 이상의 단위 테스트를 통과했고, `npm run build`로 빌드도 정상적으로 완료되었다. 주요 검증 항목은 AI 분석 엔진의 결정론성, 저장소의 데이터 복구, 내보내기 형식, 실제 과목 데이터의 역량 커버리지 등이다.

**정직성 선언.**
- 본 결과물에 AI를 활용하였으며, 이 사실을 명시한다.
- AI가 생성한 코드와 텍스트를 그대로 제출하지 않았다. 모든 AI 출력은 본인이 직접 검토하고 정확성을 확인한 뒤, 필요한 부분을 수정·보완하여 최종 반영했다.
- 출석이나 진도율을 부정한 방식으로 채우거나, 외부 스크립트를 이용한 출품 근거가 아니다.
- 수강 과목명은 본인의 HYCU 수강 이력을 기반으로 하며, 과목별 학습 내용은 본인이 직접 작성했다.
- 개인정보와 인증정보는 결과물에 포함하지 않았다.
