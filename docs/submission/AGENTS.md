# docs/submission/ — Submission Package

Contest submission archive and final package. This directory mixes editable Markdown guides with frozen binary artifacts; treat `final/` as the authoritative package surface.

## WHERE TO LOOK

| Task | Location |
|------|----------|
| Submission instructions | `final/00_제출안내.md` |
| Canonical readiness log | `final/evidence/06_빌드테스트로그.txt` |
| Current verification supplement | `final/evidence/08_현재검증보충자료.md` |
| Sample resume exports | `sample-resume-export.*`, `final/evidence/07_이력서샘플.*` |
| Screenshots | `screenshots/`, `final/evidence/04_실행화면캡처.zip` |
| Demo video | `final/evidence/05_시연영상.mp4` |

## CONVENTIONS

- `final/` filenames are numbered to match the submission order; keep numbering stable.
- `evidence/03_소스코드.zip`, screenshots ZIP, video, docx, and PDF are frozen submission artifacts unless regenerated deliberately.
- `final/evidence/06_빌드테스트로그.txt` is generated by `npm run verify:readiness`; it may record dirty worktree state.
- `npm run verify:submission` is the integrity/secret-scan gate for `final/`.
- Keep submission-facing wording Korean and concise; this directory is read by reviewers, not only developers.
- When refreshing package hashes or sizes, update `final/evidence/08_현재검증보충자료.md` with the same verification basis.

## ANTI-PATTERNS

- Do NOT treat frozen ZIP/video/docx evidence as current source without SHA/readiness revalidation.
- Do NOT put `.env`, `.git`, `node_modules`, private keys, or non-template dotenv files into source ZIPs.
- Do NOT exceed the 50 MB submission/package limit without updating the validator and docs intentionally.
- Do NOT expose personal contact data outside the official submission forms and already-prepared submission text.
