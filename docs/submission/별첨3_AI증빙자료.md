## 3. 대표 프롬프트 및 활용 결과 기록 (3개)

### 사례 1. 학습 활용 — 컴퓨터구조론 퀴즈 생성 및 오답 노트 작성

**학습 상황/목적**

컴퓨터구조론 수업 후 캐시 메모리와 파이프라인 개념이 막연하게 느껴져서, 퀴즈로 복습하고 틀린 문제는 오답 노트에 남기고 싶었다.

**입력 프롬프트**

> "캐시 메모리와 파이프라인 개념을 퀴즈 5문제로 내줘. 각 문제는 개념 이해와 적용 문제를 번갈아가며 구성해줘."

**AI 결과 요약**

AI가 개념형·적용형 문제 5문제와 정답·해설을 생성했다. 캐시 히트/미스 원리, 파이프라인 버블 발생 조건, 메모리 계층 구조 비교 등을 다루었다.

**본인 검토·수정**

- 생성된 퀴즈를 직접 풀어보고, 틀린 문제는 교재 및 강의 자료와 대조하여 오답 원인을 직접 정리했다.
- 캐시 히트/미스와 파이프라인 버블의 인과관계를 그림으로 추가 정리했다.
- AI 해설 중 부정확한 부분은 교재를 참고해 직접 수정했다.

**최종 반영 결과**

- 컴퓨터구조론 오답노트 작성 완료.
- learnprint에 "컴퓨터구조론 — 퀴즈 복습 및 오답 정리" 학습 기록으로 등록하여 역량 증거로 활용.

### 사례 2. 도구 제작 — 결정론적 역량 분류 엔진 TDD 구현

**학습 상황/목적**

흩어진 학습 경험을 역량별로 분류하는 기능이 필요했다. 네트워크 연결 없이도 동작해야 하고, 동일한 입력에는 항상 동일한 결과를 내는 결정론적 동작이 필수였다.

**입력 프롬프트**

> "학습 경험을 역량별로 분류하는 결정론적 규칙 기반 엔진을 TypeScript로 TDD 방식으로 구현해줘. 입력은 LearningItem 배열이고, 출력은 6개 역량(데이터·AI 활용, 문제해결·논리력, 협업·커뮤니케이션, 자기주도학습, 전공·직무역량, 기획·문서작성) + 기타로 그룹화해야 해. 키워드 매칭 방식으로 결정론적으로 동작해야 하고, 네트워크 호출은 절대 하면 안 돼."

**AI 결과 요약**

AI는 `LearningAnalyzer` 인터페이스와 `analyzeLearningItems` 함수를 제안했다. 각 역량에 키워드 배열을 할당하고, 입력 텍스트에서 키워드 매칭 개수로 점수를 계산한 뒤 최고점 역량에 배당하는 구조를 생성했다.

**본인 검토·수정**

- AI가 제안한 초안 키워드에서 중복과 누락을 검토하여, 한국어 학습 맥락에 맞게 키워드 사전을 재구성했다.
- "기타 학습 경험" 폴백 처리와 빈 입력 가이드 메시지를 추가했다.
- `isEmpty`, `generatedAt` 등 메타데이터 필드를 추가하여 UI에서 상태를 명확히 표현하도록 개선했다.
- 네트워크 미사용을 명시적으로 검증하는 테스트(`analyzer:uses-no-network-or-secrets`)를 추가했다.

**최종 반영 결과**

- `src/lib/ai/analyzer.ts` 구현 완료.
- 관련 테스트 5개 작성 및 전체 49개 테스트 통과.
- 네트워크 없이도 동일 입력에 동일 결과를 반환하는 결정론적 오프라인 폴백 엔진 완성.

### 사례 3. AI 에이전트 — 학습 이력 기반 역량 분석 및 자기소개 문단 생성

**학습 상황/목적**

등록한 학습 이력을 바탕으로 핵심 역량과 자기소개 문단을 작성하고, 면접 준비를 돕는 AI 에이전트 기능을 구현하고 싶었다.

**입력 프롬프트**

> "내 학습 이력으로 핵심 역량과 자기소개 문단을 작성해줘." (실제 구현에서는 BFF 서버가 학습 이력 컨텍스트를 LLM 프롬프트에 주입하여 호출한다.)

**AI 결과 요약**

서버 BFF가 학습 이력 데이터를 컨텍스트로 구성하여 LLM API에 전달했다. LLM은 6개 역량 그룹별 핵심 키워드와 자기소개 초안 문단을 생성했다.

**본인 검토·수정**

- AI가 생성한 자기소개 문단의 사실관계를 본인의 실제 학습 내용과 대조하여 검증했다.
- 과목명, 프로젝트명, 자격증명 등 구체적 증거가 정확히 인용되었는지 확인했다.
- 부정확하거나 과장된 표현은 직접 수정하여 최종 문단을 다듬었다.
- API 키는 서버 측 환경변수에만 보관하고 브라우저에는 노출되지 않도록 BFF 프록시 구조를 설계했다.

**최종 반영 결과**

- `server/agent-handler.ts` 및 관련 BFF 모듈 구현 완료.
- `/api/agent/chat` 엔드포인트를 통해 실시간 스트리밍 응답 제공.
- 역량 분석 결과와 자기소개 문단 산출.
